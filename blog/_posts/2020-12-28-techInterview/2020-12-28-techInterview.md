### 1. 객체 지향 프로그래밍

```
1. 정의
  프로그램을 수많은 '객체'라는 기본 단위로 나누고 이 객체들의 상호작용으로 서술
  - 객체란, 하나의 역할을 수행하는 '메소드와 변수(데이터)'

2. 특징
 - 인간 중심적 프로그래밍 패러다임
 - 프로그램 설계 방법론이자 개념의 일종
 - 현실 세계 사물 -> 객체
 - 어플리케이션 특징(객체의 상호작용) -> 프로그래밍(=추상화)

3. 장점
 - 코드 재사용 용이
 - 유지 보수 쉬움
 - 강한 응집력과 약한 결합력을 가진다
 - (응집력 : 기능을 수행하기 위해 얼마만큼의 연관된 책임과 아이디어가 뭉친 정도)
 - (결합력 : 프로그램의 요소가 다른 것과 얼마나 연결되어 있는지, 의존적인지)

4. 단점
 - 설계시 많은 시간 필요
 - 객체들간의 통신시 오버헤드 발생해서 처리속도가 상대적으로 느림
```

 **1. 객체 지향 프로그래밍이 뭔가요?**  
    - 프로그램 설계 방법론입니다. 현실 세계의 사물을 추상화하여 객체를 만들고, 객체간의 상호작용을 통해 시스템을 구성하는 프로그래밍 방법입니다.  
    특징으로는 추상화, 캡슐화, 상속, 다형성이 있습니다
    장점으로는 코드 재사용이 용이, 유지보수가 쉬움, 강한 응집력과 약한 결합력을 가지는 것이 있고,  
    단점으로는 설계시 많은 시간이 필요하고, 객체들간의 통신시 오버헤드가 발생해 처리속도가 상대적으로 느립니다.

**1-1. 객체지향에서 말하는 추상화가 뭔가요?**  
    - 객체에서 공통된 속성과 행위를 뽑아내는 것을 말합니다.  


**1-2. 객체지향에서 말하는 캡슐화가 뭔가요?**  
    - 변수와 함수를 하나의 단위로 묶는 것을 의미합니다. 추가로 자바에서는 접근 제어자를 통해 정보 은닉 기능을 제공합니다. 정보 은닉은, 프로그램의 세부 구현을 외부로 드러내지 않도록 감추는 것을 말합니다.  


**1-3. 객체지향에서 말하는 상속이 뭔가요?**  
    - 기존 클래스를 활용하여 새로운 클래스를 작성하는 것을 말합니다. 상속의 장점은 코드의 재사용성을 높이고 코드의 중복을 제거하여 프로그램 생산성과 유지보수에 기여합니다.    


**1-4. 객체지향에서 말하는 다형성이 뭔가요?**  
    - 다향한 형태로 나타날 수 있는 특성을 말합니다.  

**1-5. 객체지향에서 말하는 응집력과 결합력이 뭔가요?**  
    - 응집력은 기능을 수행하기 위해 얼마만큼의 책임과 아이디어가 뭉쳐있는지 나타내는 정도입니다.  
    - 결합력은 다른 요소와 얼마나 의존적인지 나타내는 정도입니다.

**2. 객체와 인스턴스가 뭔가요?**  
    - 인스턴스는 객체를 실체화한 것입니다. 즉, 프로그램 코드 상에서 자료형이 임의의 클래스로 선언된 클래스를 객체라 하고, 코드 컴파일 후 프로그램이 실행될 때 해당 객체가 메모리에 적재되면 인스턴스라 합니다. 

**3. 객체 지향 프로그래밍과 절차지향 프로그래밍을 설명해주세요**
    - 절차지향 프로그래밍이란 '프로그램이 무슨일을 하는가'의 관점으로 바라본 프로그래밍 기법이고,
    - 객체지향 프로그래밍은 '어떤 대상을 모델링하는가'의 관점으로 바라본 프로그래밍 기법입니다.
    - 즉, 절차지향 프로그래밍은 프로그램의 순서와 흐름을 먼저 세우고 필요한 자료구조와 함수들을 설계하는 방식이고, 객체지향 프로그래밍은 반대로 자료구조와 이를 중심으로 한 모듈들을 먼저 설계한 다음에 이들의 실행순서와 흐름을 짜는 방식이다.

### 2. RESTful API

```
1. 정의
 - 소프트웨어 아키텍처의 한 형식으로 자원을 정의하고 자원에 대한 주소를 지정하는 방법  

2. 특징
 - 지정한 주소를 HTTP Method를 통해 자원을 처리한다.
 - REST의 6가지 기본 원칙을 성실히 지킨 서비스 디자인을 RESTful하다라고 표현

3. 장점
 - 기존 웹 인프라를 그대로 사용할 수 있다.
 - 쉽고, 빠르다
 - 세부적인 표현 문구를 JSON, XML등 다양한 언어를 이용하여 작성할 수 있다.

4. 단점
 - 메소드 형태가 제한적이다.
 - 표준 규약이 없다.
 - HTTP 통신 모델에 대해서만 지원합니다.
```

**1. API란 무엇인가요?**  
    - API란, 응용 프로그램에서 사용할 수 있도록 운영 체제나 프로그래밍 언어가 제공하는 기능을 제어할 수 있게 만든 인터페이스를 뜻합니다.  

**2. REST API란 무엇인가요?**  
    - 자원이 있는 곳을 URI로 정의하고, 자원을 이용하는 방법을 HTTP Method를 통해 자원을 처리하는 것을 말합니다.  

**3. RESTful 하게 API를 디자인 한다는 것은 무엇을 의미하는가?**  
    - REST API의 6가지 기본 원칙을 성실히 지킨 서비스를 RESTful하다라고 표현합니다.  

### 3. TDD

```
1. 정의
 - 테스트 주도 개발
 - (개발 테스트 단위가 함수단위로 매우 작아서 작성하는 거의 모든 함수가 테스트에 포함된다.)

2. 관련 용어
 1) BDD : 시나리오 기반으로 테스트 케이스 작성
  - Given -> When -> Then
  - 어떤 상태 (Given) -> 어떤 상태 변화를 가했을 때(When) -> 기대하는 상태로 완료(Then)

3. 장점
 - 피드백이 많아진다.
 - (테스트를 통과하는 것으로 개발이 잘 진행되는지 자주 확인가능해진다.)
 - 코드 복잡도가 낮아진다.

4. 단점
 - 개발시간이 길어진다.
```
